# AI 기반 소셜 미디어 콘텐츠 생성 워크플로우

이 워크플로우는 Notion 데이터베이스에 새 항목이 생성되면, 입력된 주제와 메시지 톤을 바탕으로 플랫폼별 텍스트 콘텐츠와 대표 이미지를 생성해 같은 Notion 페이지 본문에 저장한다.

## 전체 흐름

1. Notion의 `컨텐츠 생성 DB`에서 새 항목 생성을 감지한다.
2. 새 항목의 `주제/키워드`와 `메세지 톤`을 읽어 공통 콘텐츠 브리프를 만든다.
3. `메세지 톤`이 `친근함`이면 친근형 콘텐츠 생성 흐름을 탄다.
4. 그 외 톤은 전문형 콘텐츠 생성 흐름을 탄다.
5. 선택된 톤에 맞춰 Instagram, 네이버 블로그, Threads용 콘텐츠를 각각 생성한다.
6. 플랫폼별 생성 결과를 하나로 모아 Notion 페이지 본문에 저장한다.
7. 콘텐츠 브리프를 바탕으로 대표 이미지 프롬프트를 만들고 이미지를 생성한다.
8. 생성된 이미지를 Cloudinary에 업로드한 뒤, 이미지와 URL을 Notion 페이지 본문에 추가한다.

## 단계별 역할

| 단계 | 역할 |
| --- | --- |
| 응답 감지 | Notion 데이터 소스에서 새로 생성된 항목을 감지한다. 한 번에 최대 5개 항목을 처리한다. |
| 컨텐츠 브리프 생성 | `주제/키워드`와 `메세지 톤`을 입력으로 받아 플랫폼 공통 기준이 되는 JSON 브리프를 만든다. |
| 톤 분기 | Notion 항목의 `메세지 톤` 이름이 `친근함`인지 확인해 친근형 또는 전문형 생성 흐름으로 나눈다. |
| 친근형 콘텐츠 생성 | 친근한 톤으로 Instagram 캡션, 네이버 블로그 글, Threads 게시글을 각각 생성한다. |
| 전문형 콘텐츠 생성 | 전문적인 톤으로 Instagram 캡션, 네이버 블로그 글, Threads 게시글을 각각 생성한다. |
| 결과 병합 | 톤 분기에서 생성된 플랫폼별 결과를 `인스타그램`, `네이버 블로그`, `스레드` 항목으로 모은다. |
| 플랫폼별 컨텐츠 저장 | 콘텐츠 브리프와 플랫폼별 생성 결과를 원본 Notion 페이지 본문에 추가한다. |
| 대표 이미지 프롬프트 생성 | 콘텐츠 브리프만 기준으로 이미지 생성 AI에 전달할 영어 묘사 프롬프트를 만든다. |
| 대표 이미지 생성 | Cloudflare AI 이미지 생성 API에 프롬프트를 보내 대표 이미지를 만든다. |
| Cloudinary 업로드 | 생성된 이미지를 PNG로 변환해 Cloudinary의 `ai-social-media-content-generator` 폴더에 업로드한다. |
| 대표 이미지 본문 추가 | Cloudinary 이미지 URL을 사용해 Notion 페이지에 대표 이미지와 링크를 추가한다. |

## 콘텐츠 생성 방식

### 공통 브리프

공통 브리프는 모든 플랫폼 콘텐츠의 기준이다. 출력은 JSON이며 `기본정보`, `주제와 기획방향`, `예상독자와 방침`, `메시지 구성`, `톤앤매너`를 포함한다.

브리프는 입력값을 벗어난 새로운 사실, 수치, 효능, 근거를 임의로 추가하지 않도록 설계되어 있다.

### 플랫폼별 콘텐츠

각 플랫폼 콘텐츠는 공통 브리프를 입력으로 받아 `platform`과 `content`만 가진 JSON으로 생성된다.

| 플랫폼 | 생성 내용 |
| --- | --- |
| Instagram | 150~500자 캡션을 생성한다. 본문, CTA, 해시태그 5개를 포함한다. |
| 네이버 블로그 | Markdown 형식의 블로그 글을 생성한다. 제목, 도입부, 4~6개 소제목, 요약과 CTA, 해시태그를 포함한다. |
| Threads | 150~350자를 권장하고 최대 500자를 넘지 않는 게시글을 생성한다. 해시태그는 0~3개까지 사용할 수 있다. |

친근형은 더 쉬운 표현과 대화형 문체를 사용하고, 전문형은 구조적이고 신뢰감 있는 경어체를 사용한다. 두 흐름 모두 브리프에 없는 주장이나 검증되지 않은 사실을 추가하지 않는 규칙을 공유한다.

## Notion 저장 구조

플랫폼별 텍스트 생성이 끝나면 원본 Notion 페이지 본문에 아래 순서로 블록이 추가된다.

1. `컨텐츠 브리프`
2. 브리프 JSON 코드 블록
3. `인스타그램`
4. Instagram 결과 JSON 코드 블록
5. `네이버 블로그`
6. 네이버 블로그 결과 JSON 코드 블록
7. `스레드`
8. Threads 결과 JSON 코드 블록

네이버 블로그 결과는 길이가 길 수 있어 Notion 본문에 넣을 때 1,900자 기준으로 나누어 저장한다.

## 대표 이미지 처리

대표 이미지는 플랫폼별 콘텐츠가 아니라 공통 브리프를 기준으로 만든다.

1. 브리프의 주제, 핵심 메시지, 예상 독자, 포함할 내용, 톤앤매너를 종합해 영어 이미지 프롬프트를 생성한다.
2. 해당 프롬프트를 Cloudflare AI 이미지 생성 API로 보낸다.
3. 응답으로 받은 base64 이미지를 PNG 바이너리로 변환한다.
4. Cloudinary에 `notion_{{Notion 페이지 ID}}` 형식의 public ID로 업로드한다.
5. 업로드된 `secure_url`을 Notion 페이지에 외부 이미지 블록과 링크 문단으로 추가한다.

`workflow.json`에는 Cloudflare 계정 ID와 API 토큰이 템플릿 값으로 들어 있으므로, 실제 실행 환경에서는 해당 값이 유효한 값으로 설정되어 있어야 한다.
